# TourneyEdge AI 🏀
### NCAA Tournament Betting Analysis Platform

A production-ready, AI-powered betting analysis tool for the NCAA men's tournament. Compare any two teams side-by-side, view advanced analytics, current betting lines, and get a structured model prediction with confidence scoring.

> ⚠️ **Disclaimer**: For informational and entertainment purposes only. Not financial, betting, or investment advice.

## Quick Start (Local Dev)

```bash
# 1. Install dependencies
npm install

# 2. Copy env file
cp .env.example .env.local

# 3. Run in demo mode (no API keys needed)
npm run dev

# 4. Open http://localhost:3000
```

In demo mode (`DATA_MODE=mock`), all data comes from the seed file in `src/lib/mock-data.ts`. No API keys required.

## Wiring Up Real APIs

### Step 1 — The Odds API (Betting Lines)

The easiest and most important integration. Gets live spreads, totals, and moneylines from 40+ sportsbooks.

1. Sign up at **https://the-odds-api.com** (free tier: 500 requests/month)
2. Copy your API key from the dashboard
3. Add to `.env.local`:

```env
ODDS_API_KEY=your_key_here
ODDS_API_BOOK=draftkings        # or fanduel, betmgm, caesars
ODDS_API_SPORT=basketball_ncaab
DATA_MODE=live
```

4. That's it. The `/api/odds` route will now return live lines.

**Cost**: Free tier covers ~16 requests/day. Paid plans start at $0/month for 500/month up to $79/month for unlimited.

---

### Step 2 — KenPom (Advanced Metrics)

KenPom provides the best adjusted efficiency metrics (AdjOE, AdjDE, tempo, luck) used by virtually every serious college basketball model.

1. Subscribe at **https://kenpom.com** (~$20/year)
2. Add to `.env.local`:

```env
KENPOM_EMAIL=your@email.com
KENPOM_PASSWORD=yourpassword
DATA_MODE=live
```

3. The adapter in `src/lib/adapters/kenpom.ts` authenticates, fetches the ratings table, parses it, and merges the data into each team object.

**How it works**: KenPom has no official API. The adapter logs in via POST to get a session cookie, then fetches `kenpom.com/index.php` and parses the `#ratings-table` HTML table. This is the standard approach used by analytics tools — KenPom is aware subscribers do this for personal use.

**Fallback**: If KenPom is unavailable or credentials are wrong, the app gracefully falls back to the mock efficiency values.

---

### Step 3 — ESPN (Scoring / Record Stats)

ESPN has an unofficial but stable public API — no key required.

```env
USE_ESPN_PUBLIC=true
DATA_MODE=live
```

The adapter in `src/lib/adapters/espn.ts` fetches scoring averages and records for each team using their ESPN team IDs (see `ESPN_TEAM_IDS` in that file — add any teams missing from the map).

---

### Step 4 — OpenAI (AI Narrative, Optional)

Adds a plain-English explanation paragraph to each analysis, generated by GPT.

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini   # cheap and fast; use gpt-4o for better quality
DATA_MODE=live
```

The rules-based model runs with or without this. OpenAI is only called if the key is present.

**Cost**: gpt-4o-mini is ~$0.00015/1K tokens. Each analysis call uses ~300 tokens = less than $0.0001 per analysis.

---

### Step 5 — Sportradar (Premium, Optional)

For production-grade stats including play-by-play, rosters, and injury reports:

1. Sign up at **https://developer.sportradar.com** (trial available)
2. Add:

```env
SPORTRADAR_API_KEY=your_key_here
```

3. Add a `sportradar.ts` adapter in `src/lib/adapters/` following the same pattern as `espn.ts`.

## Adding More Teams

Edit `src/lib/mock-data.ts` and add entries to the `MOCK_TEAMS` array. Each team needs:

```typescript
{
  id: 'unique_id',        // lowercase, no spaces
  name: 'Full Name',
  shortName: 'Short',
  seed: 1,                // 1–16
  region: 'East',         // East | West | South | Midwest
  conf: 'ACC',
  color: '#hex',
  emoji: '🔵',
  // Stats — replace with real values or wire KenPom

  dataSource: 'mock',
```

## Caching (Redis / Upstash)

For production, add Redis caching to avoid hammering paid APIs:

1. Create a free Redis instance at **https://upstash.com**
2. Add to `.env.local`:

```env
UPSTASH_REDIS_REST_URL=https://your-instance.upstash.io
UPSTASH_REDIS_REST_TOKEN=your_token
CACHE_TTL_SECONDS=300
```

3. Install: `npm install @upstash/redis`
4. Wrap API calls in the adapters with:

```typescript
import { Redis } from '@upstash/redis';
const redis = new Redis({ url: process.env.UPSTASH_REDIS_REST_URL!, token: process.env.UPSTASH_REDIS_REST_TOKEN! });

const cached = await redis.get(cacheKey);
if (cached) return cached;
// ... fetch ...
await redis.set(cacheKey, result, { ex: parseInt(process.env.CACHE_TTL_SECONDS ?? '300') });
```

## Environment Variables Reference

| Variable | Required | Description |
|---|---|---|
| `DATA_MODE` | Yes | `mock` or `live` |
| `KENPOM_EMAIL` | For KenPom | Your kenpom.com email |
| `KENPOM_PASSWORD` | For KenPom | Your kenpom.com password |
| `ODDS_API_KEY` | For live odds | From the-odds-api.com |
| `ODDS_API_BOOK` | No | Preferred book (default: draftkings) |
| `ODDS_API_SPORT` | No | Default: basketball_ncaab |
| `USE_ESPN_PUBLIC` | No | `true` to enable ESPN stats |
| `OPENAI_API_KEY` | No | For AI narrative text |
| `OPENAI_MODEL` | No | Default: gpt-4o-mini |
| `UPSTASH_REDIS_REST_URL` | No | Redis cache URL |
| `UPSTASH_REDIS_REST_TOKEN` | No | Redis auth token |
| `CACHE_TTL_SECONDS` | No | Default: 300 (5 min) |
